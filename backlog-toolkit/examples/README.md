# Examples Directory

This directory contains filled-out examples of each template to help you understand how to use them effectively.

## What's Included

- **feature-request-example.md** - A complete feature request example
- **bug-fix-example.md** - A complete bug fix example
- **sprint-planning-example.md** - A complete sprint planning example
- **product-backlog-example.md** - A sample product backlog table
- **git-commit-example.md** - Examples of git commit messages
- **workflow-diagrams-example.md** - Mermaid.js workflow diagrams for backlog management

## How to Use These Examples

1. **Review the examples** to understand the expected format and level of detail
2. **Copy the template** (not the example) when creating new items
3. **Use the examples as reference** when you're unsure what to include
4. **Adapt the examples** to match your project's needs and structure

## Notes

- All examples use generic/placeholder data (no project-specific information)
- Examples demonstrate best practices for writing clear, actionable backlog items
- File paths and references are examples only - adjust to match your project structure
- ID formats (FR-XXX, BF-XXX) can be customized to match your conventions

## Best Practices Demonstrated

- Clear user stories following "As a... I want... So that..." format
- Specific, testable acceptance criteria
- Well-documented technical references
- Proper priority and story point estimation
- Complete environment details for bug reports
- Comprehensive task breakdowns for sprint planning
